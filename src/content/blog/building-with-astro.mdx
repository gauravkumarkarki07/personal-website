## Why Astro?

When I decided to rebuild my personal website, I wanted something that was:
- **Fast** - Zero JavaScript by default, only add what's necessary
- **Developer-friendly** - Easy to write content and maintain
- **Modern** - Support for the latest web technologies
- **Flexible** - Room to grow as I add new features

Astro checked all these boxes and more.

## The Architecture

### Content Collections

One of Astro's killer features is Content Collections. Instead of managing blog posts and projects in a database or CMS, I can write them in Markdown/MDX files with type-safe frontmatter.

```typescript
const blogCollection = defineCollection({
  type: "content",
  schema: z.object({
    title: z.string(),
    description: z.string(),
    publishDate: z.date(),
    tags: z.array(z.string()),
    draft: z.boolean().default(false),
  }),
});
```

This gives me:
- Type safety at build time
- Auto-completion in my editor
- Validation errors if I mess up frontmatter
- Easy querying with `getCollection()`

### Component-Driven Design

I built reusable components for common patterns:
- `AnimatedSection` - Scroll-triggered fade-in animations
- `BlogCard` - Preview cards for blog posts
- `ProjectCard` - Showcase project thumbnails with tech stack

Each component is self-contained and easy to test.

### Styling with Tailwind CSS

Tailwind CSS 4.0 made styling a breeze. I created custom design tokens:

```css
@theme {
  --color-background: #1e201f;
  --color-foreground: #fcfcfe;
  --color-accent: #60a5fa;
}
```

This ensures consistency across the entire site while maintaining flexibility.

## Performance Results

The results speak for themselves:
- **Lighthouse Score**: 100/100 across all metrics
- **First Contentful Paint**: < 0.5s
- **Time to Interactive**: < 1s
- **Bundle Size**: < 50KB JavaScript

All thanks to Astro's static site generation and minimal JavaScript approach.

## Lessons Learned

1. **Content Collections are powerful** - They make managing content feel natural for developers
2. **View Transitions are magical** - Smooth page transitions with minimal code
3. **TypeScript everywhere** - Type safety from content to components prevents bugs
4. **Progressive enhancement** - Start with zero JS, add interactivity where needed
