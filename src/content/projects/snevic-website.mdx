## Project Overview

Snevic is a non-profit organization based in Melbourne that needed a modern web presence with the ability to manage content dynamically. I built a comprehensive solution using Next.js for the frontend and Payload CMS for content management.

## Key Features

### Dynamic Content Management
- **Page Builder**: Custom page creation with flexible content blocks
- **Blog System**: Full-featured blog with categories, tags, and author management
- **Event Management**: Create and manage community events with RSVP functionality
- **Media Library**: Organized asset management with image optimization

### User Features
- **Membership Registration**: Custom registration flow with email verification
- **User Dashboard**: Members can view their profile and event history
- **Newsletter Subscription**: Integrated email newsletter system
- **Contact Forms**: Multiple contact forms for different inquiries

### Admin Panel
- **Content Editors**: Rich text editing with markdown support
- **User Management**: Role-based access control (Admin, Editor, Member)
- **Analytics Dashboard**: Track website metrics and user engagement
- **SEO Controls**: Meta tags, Open Graph, and sitemap generation

## Technical Implementation

### Frontend Architecture

Built with **Next.js 14** using the App Router:

```typescript
// Dynamic page routing
export async function generateStaticParams() {
  const pages = await payload.find({
    collection: 'pages',
    limit: 100,
  });

  return pages.docs.map((page) => ({
    slug: page.slug,
  }));
}
```

Key technologies:
- **React Server Components** for optimal performance
- **TypeScript** for type safety across the stack
- **Tailwind CSS** for responsive, utility-first styling
- **Next.js Image** for automatic image optimization

### Content Management

Implemented **Payload CMS** as a headless CMS:

```typescript
// Custom collection example
export const Events: CollectionConfig = {
  slug: 'events',
  admin: {
    useAsTitle: 'title',
    defaultColumns: ['title', 'date', 'status'],
  },
  fields: [
    {
      name: 'title',
      type: 'text',
      required: true,
    },
    {
      name: 'date',
      type: 'date',
      required: true,
    },
    {
      name: 'description',
      type: 'richText',
    },
    {
      name: 'status',
      type: 'select',
      options: ['upcoming', 'ongoing', 'completed'],
    },
  ],
};
```

Benefits:
- **Type-safe APIs**: Auto-generated TypeScript types
- **Flexible schema**: Easy to add new content types
- **Built-in authentication**: Secure admin access
- **Relationships**: Link content across collections

## Challenges & Solutions

### Challenge 1: Dynamic Page Generation
**Problem**: Need to support various page layouts without hardcoding templates.

**Solution**: Implemented a flexible blocks system where each page is composed of reusable content blocks (Hero, Text, Images, CTA, etc.). Editors can drag-and-drop blocks to create custom layouts.

### Challenge 2: Performance with Rich Content
**Problem**: Pages with many images and rich content were slow to load.

**Solution**:
- Implemented Next.js Image component with automatic optimization
- Used incremental static regeneration (ISR) for frequently updated pages
- Added lazy loading for below-the-fold content

### Challenge 3: User Authentication
**Problem**: Needed member authentication without over-engineering.

**Solution**: Leveraged Payload's built-in authentication with custom fields for member-specific data. Extended with email verification flow.

## Lessons Learned

1. **Headless CMS is powerful** - Payload CMS provided the perfect balance of flexibility and ease of use
2. **Type safety matters** - TypeScript prevented numerous runtime errors during development
3. **Performance by default** - Next.js's built-in optimizations made the site fast without extra effort
4. **User feedback is crucial** - Regular check-ins with the Snevic team led to better features
