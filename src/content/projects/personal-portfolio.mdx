## Project Overview

This portfolio website showcases my work as a software engineer while demonstrating modern web development practices. Built with Astro, it features smooth animations, a blog system, project showcases, and an integrated resume viewer.

## Design Philosophy

### Minimalist with Motion

I wanted a design that was:
- **Clean and spacious** - generous whitespace, clear hierarchy
- **Interactive** - smooth animations without being distracting
- **Fast** - optimized for performance and quick loading
- **Maintainable** - easy to add content and make changes

The result is a "Minimalist with Motion" approach: clean layouts enhanced with subtle scroll animations and hover effects.

## Key Features

### Content Management
- **Blog System**: Write posts in MDX with full markdown support
- **Project Showcases**: Detailed project pages with tech stacks
- **Resume Viewer**: Embedded PDF viewer with download option

### Interactive Elements
- **Scroll Animations**: Elements fade in as you scroll

## Technical Implementation

### Animation System

Created a reusable `AnimatedSection` component using Intersection Observer:

```astro
---
export interface Props {
  delay?: number;
  class?: string;
}
const { delay = 0, class: className } = Astro.props;
---

<div
  class:list={["animate-section opacity-0", className]}
  style={`animation-delay: ${delay}ms`}
>
  <slot />
</div>

<script>
  const observerOptions = {
    threshold: 0.1,
    rootMargin: "0px 0px -50px 0px",
  };

  const observer = new IntersectionObserver((entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add("animate-fadeInUp");
        observer.unobserve(entry.target);
      }
    });
  }, observerOptions);

  document.querySelectorAll(".animate-section").forEach((el) => {
    observer.observe(el);
  });
</script>
```

This provides:
- Performant animations using Intersection Observer
- Customizable animation delays
- Progressive enhancement (no JS = content still visible)
- Respects `prefers-reduced-motion`

Previous/Next navigation between posts:

```typescript
const allPosts = await getCollection("blog", ({ data }) => !data.draft);
const sortedPosts = allPosts.sort(
  (a, b) => b.data.publishDate.getTime() - a.data.publishDate.getTime()
);
const currentIndex = sortedPosts.findIndex((p) => p.slug === post.slug);
const prevPost = currentIndex < sortedPosts.length - 1 ? sortedPosts[currentIndex + 1] : null;
const nextPost = currentIndex > 0 ? sortedPosts[currentIndex - 1] : null;
```

## Challenges & Solutions

### Challenge 1: Animation Performance
**Problem**: Too many animations caused janky scrolling on mobile.

**Solution**:
- Used Intersection Observer instead of scroll listeners
- Limited animations to transform and opacity (GPU-accelerated)
- Added `will-change` hints for smooth animations

### Challenge 2: Resume PDF Display
**Problem**: PDF viewing isn't consistent across browsers.

**Solution**:
- Used iframe with fallback messages
- Provided prominent download button
- Tested across Chrome, Safari, Firefox, and mobile browsers

### Challenge 3: Type Safety Across Content
**Problem**: Easy to make mistakes in frontmatter.

**Solution**:
- Used Zod schemas in Content Collections
- Build fails if content doesn't match schema
- Auto-completion in editor for frontmatter

## Lessons Learned

1. **Start with content structure** - Defining schemas early prevented refactoring later
2. **Progressive enhancement** - Core content works without JS, enhanced with animations
3. **Design systems scale** - Custom properties and utility classes kept styles consistent
4. **Performance by default** - Astro's zero-JS approach made optimization easy
